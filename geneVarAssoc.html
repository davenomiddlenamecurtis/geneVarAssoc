geneVarAssoc

geneVarAssoc is a utility to set up analyses with scoreassoc, which is documented separately. scoreassoc is available at: github.com/davenomiddlenamecurtis/scoreassoc

or: http://www.davecurtis.net/software/dcurtis/scoreassoc.html

There is no formal published account of geneVarAssoc but the actions it performs were described in this publication, which could be cited:

Practical experience of the application of a weighted burden est to whole exome sequence data for obesity and schizophrenia. D Curtis and the UK10K Consortium. Ann Hum Genet 80, 38-49. 2016.

Broadly, geneVarAssoc takes vcf files for cases and controls, uses tabix to extract variants for a gene, produces the necessary input files for scoreassoc and then runs scoreassoc.

Until recently, geneVarAssoc has only been used in house and this documentation remains work in progress.

Compilation

Change to the src folder and then run:

make –f geneVarAssoc.mak

This command should produce executables called geneVarAssoc and intVarAssoc, which should then be copied to the directory assigned to DCBIN in geneVarAssoc.mak, by default called ../bin.

Solution and project files are provided to allow the program to be compiled with Microsoft Visual Studio. To compile for MSDOS/Windows, the preprocessor variable MSDOS must be defined.

Running geneVarAssoc

Typical usage:

geneVarAssoc --arg-file ../pars/gva.SSS.ct08.rare.arg --gene DRD2

The arguments for the program can be specified on the command line or in a file or using a combination of both. The arguments are in pairs, with the first member of the pair, starting with --, specifying the option to be set and the second member the value for that option.

When vcf file names are specified, if they contain the * character then this means that there are different vcf files for each chromosome and the * will be substituted with the chromosome number to extract the relevant variants for a particular gene.

For the simplest usage, there will be one or more vcf files containing control genotypes and one or more vcf files containing case genotypes. However one may also work with only allele frequencies, rather than genotypes, for controls and/or cases. If the control and case genotypes are in the same vcf file then the phenotypes are specified with a  separate file. If the subjects consist of trios then a separate file is used to define the pedigree relationships.

In order to run geneVarAssoc you will need the following:

A set of bgzipped vcf files which have been indexed with tabix and accompanied by their vcf.gz.tbi files. Case and control files can be separate and there can be up to ten case and ten control files. If separate case and control files are not used then a single file can hold all the subjects, in which case a separate file specifying phenotypes must be provided. If the subjects consist of sets of trios then no phenotypes need be provided and it is assumed that the children are all affected. Generally the vcf files will contain genotypes but they can instead contain genotype probabilities as output by IMPUTE2. If individual level data is not available then instead a file of allele frequencies for cases or controls or both can be provided, e.g. from 1KG or ExAC. If this is done, the number of subjects used to produce these frequencies must also be given. There can be a separate vcf file for each chromosome or else all chromosomes can be combined.

 

The human reference sequence must be provided in files named chr1.fa, chr2.fa, etc.

 

In the same folder must be a file containing the coordinates of transcripts in refseq format and sorted by gene so that all the transcripts for each gene are grouped together, as in the provided example file refseqgenesorted101115.txt.

 

The programs tabix and scoreassoc must both be on the PATH.

When geneVarAssoc is run then it uses tabix to extract the subset of variants matching the gene coordinates. Then it annotates them. Then it produces scoreassoc data files for valid variants and produces weights according to the annotations. Then it runs scoreassoc.

Typically a parameter file specified by --arg-file would contain the specifications for the analyses and the locations of the data and reference files and then the --gene argument would specify the gene to be analysed. Two example parameter files are provided. In exampleContFreqFile.arg, a 1KG file is used to provide allele frequencies based on 195 subjects and the case genotypes would be in files called BP_chr1.vcf.gz, BP_chr2.vcf.gz, etc. In gva.SSS.ct08.rare.arg, all the case and control genotypes are in scz-only-exome-all-nozeros.vcf.gz and the phenotypes are specified in SSSPhenotypes.txt.

Because of the identifiable nature of sequence data, it is not possible to provide example data files.

When trying to get the program running, one can inspect the temporary files which are produced in order to elucidate problems. The most frequent reason for tabix to fail to extract any variants is a mismatch between the --add-chr variable and the way variants are named in the vcf file. That is, if the vcf file refers to a chromosome as e.g. chr10 then one must use --add-chr 1.

Here is the list of options:

--arg-file filename

Specifies the name of a file from which to read additional arguments. Once this file has been read further options will be read from the current input source, whether it be the command line or another file. Thus, argument files can be nested and/or chained. The simplest way to run the program is to put all the arguments in a single file (e.g. options.txt) and then to enter:

geneVarAssoc --arg-file options.txt --gene DRD2

--ref-gene-file filename

Specifies a file in refseq format, with first line beginning:

#bin    name    chrom   strand  txStart . . . 

The file must have been sorted on the gene name to bring all transcripts for each gene together. E.g. refseqgenesorted101115.txt, which is provided in the exampleFiles directory.

--reference-path pathname

Provides the name of the folder containing the reference genome files, needed if the inbuilt annotation functions are to be used, having names such as chr17.fa, etc.

--cont-file filename

Provides the name of a vcf file containing control subject genotypes. There can be more than one of these.

--case-file filename

Provides the name of a vcf file containing case subject genotypes. There can be more than one of these. If case and control subjects are in the same file then this should be specified as the case file and the phenotype status should be specified using the --phenotype-file option. If trios are used then it is assumed that the child in each trio is affected and the vcf file containing all subjects is specified using --case-file while the trio structure is defined using --trio-file.

--phenotype-file filename

Specifies the name of a file which provides phenotypes of the subjects in the vcf file(s). There is one row per subject, consisting of either 0 or 1, for case or control, or any other number, in which case the subject will be omitted from the analysis. The rows must exactly match the order of subjects in the vcf file(s). This option can be used if one does not have separate vcf files for cases and controls. Typically one would use it with just a single vcf file. However, if for some reason one does wish to use this option with multiple vcf files please note that any control files will be read in before any case files, regardless of which are specified first. So the phenotype file must have phenotypes for the subjects in the control files before those in the case files. This aside, the vcf files are read in in the order in which they are specified. The phenotypes in this file will then override the control or case file specification. (A possible application of this would be if one wished to perform some kind of permutation testing.)

--ID-and-phenotype-file filename

Specifies the name of a file which provides phenotypes of the subjects in the vcf file(s). There is one row per subject, consisting of the subject ID as in the vcf file followed by a space then a 0 or 1 (for case or control) or any other number (to be ignored). The order does not need to match the order in the vcf file(s). If a subject has phenotype other than 0 or 1 or if there is no entry for a subject then it will not be included in analysis.

--use-probs

Specifies that the vcf file contains genotype probabilities as output by IMPUTE2 rather than genotypes.

--samples-file filename

Specifies the name of a file which provides phenotypes in the format used by IMPUTE2. The program looks for a column whose name begins PH (case sensitive) and then reads phenotypes (0 or 1) from that column. The order must match that of the subjects in the vcf file(s), as described for the --phenotype-file option.

--trio-file filename

Indicates that the subjects consist of sets of trios as defined in the file specified. Each subject to be used occupies one row. The first entry is the ID of the subject, as in the vcf file. If the subject has parents then their IDs are given next, father then mother. If the subject is a parent then the parental IDs are given as 0 0. Subjects in the vcf file but not in this file will be ignored.

--cont-freq-file filename

Specifies a vcf file providing the allele frequencies of variants in controls. If this option is used there will be no vcf file for control genotypes.

--num-cont

Specifies the number of controls which had been used to obtain the control allele frequencies, i.e. the total number of controls genotyped. The number of successful genotypes may vary slightly between variants but the maximum number should be given here.

--case-freq-file

Specifies a vcf file providing the allele frequencies of variants in cases. If this option is used there will be no vcf file for case genotypes.

--num-case

Specifies the number of cases used to obtain the case allele frequencies, i.e. the total number of cases genotyped. The number of successful genotypes may vary slightly between variants but the maximum number should be given here.

--gene name

Provide the name of the gene to be extracted and analysed.

--allele-freq-str string

String used to recognise the allele frequency entry in a vcf file which is used to provide allele frequencies for cases or controls. If the vcf file contains entries such as …;EUR_AF=0.42;… then one would specify --allele-freq-str EUR_AF

--allele-number-str str

String used to recognise the allele number entry in a vcf file which is used to provide allele frequencies for cases or controls. If the vcf file contains entries such as …;EUR_AN=2414;… then one would specify --allele-number-str EUR_AN

--allele-count-str str

String used to recognise the alternate allele count entry in a vcf file which is used to provide allele frequencies for cases or controls. If the vcf file contains entries such as …;EUR_AC=27;… then one would specify --allele-number-str EUR_AC. If there is no allele frequency field then the frequency will be calculated as the count divided by the number (as in ExAC).

--bait-file filename

File containing coordinates for the baits. If this is specified then they will be used to determine the coordinates of exomes to extract rather than relying only on the gene coordinates in the refseq file.

--upstream number

Number of bases upstream of the gene to extract.

--downstream number

Number of bases downstream of the gene to extract.

--margin number

Number of additional bases around the baits to extract.

--weight-factor wf

Weight factor to provide to scoreassoc for weighting based on allele frequency.

--use-ensembl 0 or 1

If 1, use ensembl to carry out annotation, otherwise use the program’s own internal annotation routines. In order to use this, you must have the perl version of variant effect predictor (VEP) set up so that commands such as the following can run:

perl variant_effect_predictor.pl -i predictorQuery.txt -o predictorOutput.txt --most_severe

--consequence-threshold num

Tell scoreassoc to only use variants with an annotation number of num or higher (nonsynonymous=8).

--use-consequence-weights 0 or 1

If 1, provide scoreassoc with functional weights for each variant, such that variants with predicted effect on function are weighted more highly.

--only-use-SNPs 0 or 1

If 1, only use variants with two alleles, each consisting of a single base. Do not use indels.

--write-comments 0 or 1

If 1, provide information about each variant to scoreassoc to be output as comments.

--write-score-file 0 or 1

If 1, instruct scoreassoc to output scores as well as standard output file.

--do-recessive-test 0 or 1

If 1, instruct scoreassoc to test for recessive effects.

--weight-threshold threshold

Instructs scoreassoc to only use those variants whose weight (based on rarity and function) exceeds the threshold.

--LD-threshold threshold

Instructs scoreassoc to drop a variant which occurs in more than the stated proportion of subjects with another variant.

--add-chr 0 or 1 or 0011001, etc.

If 1, adds CHR to the chromosome number when running tabix to extract variants from all the vcf files. If 0, adds CHR for none of them. If a sequence of 0's and 1's, equal in length to the number of vcf files, then adds CHR to the vcf files corresponding to the 1's.

--GQ-threshold threshold

Only output genotypes reaching this threshold for the genotype quality score and treat others as unknown.

--unknown-if-untyped 0 or 1

If 1, treat a variant which has no entry in a vcf file as having unknown genotype for all subjects in that file. If 0, treat all subjects in that file as being wild type. If 1 and a frequency file is used then any variants missing from the frequency file will be omitted from the analysis. If 0 they will be treated as having frequency of 0 (i.e. all wild type). The assumption then is that the variant was adequately covered but did not appear in the vcf file because no alt alleles were seen. Default value is 0.

--wild-if-unknown 0 or 1

If 1, treat all unknown genotypes as wildtype. This can be useful if a number of vcf files have been merged and variants missing from one or more of the original files have been coded as unknown. However, doing this means one discards information about genotypes which really should be called as unknown because sufficient information was not available to call them reliably. Default value is 0.

--unknown-if-no-pass 0 or 1

If 1, variants without a PASS will be treated as unknown for that vcf file. If 0, the genotypes will be used. Default value is 1.

--skip-if-no-pass 0 or 1

If 1, variants without a PASS will be omitted rather than being treated as unknown. Default value is 0.

--ignore-alleles 0 or 1

If 1, treat variants at the same position but with possibly different sets of alleles as being the same variant. Default value is 0.

--use-haplotypes 0 or 1

If 1, instruct scoreassoc to treat the genotypes as phased for recessive analyses. This is appropriate if the alleles are ordered within the genotypes or if trios are used. Default value is 0.

--dont-extract-gene 0 or 1

If 1, skip over the step of extracting the variants from the vcf file and assume that the necessary files have already been created. Used for debugging. Default value is 0.

--keep-temp-files 0 or 1

If 1, do not delete the temporary vcf files which are produced. This means one can examine them subsequently, for example to see which variants they contain, and can be useful for debugging. Default value is 0.

--do-not-run 0 or 1

If 1, set up all the files for the analysis but do not actually run scoreassoc. A bash script or, under MSDOS, a batch file will be created which can be run to perform the analysis. Can be useful for debugging. Default value is 0.

--sequence-path pathname

This can be used to provide the name of the folder containing the main vcf files. If this is specified, then rather than providing the full path for each vcf file one would specify just the file name. This potentially allows different analyses to look for the same files in different places and is intended to facilitate using scratch space, etc.

--clear-cont 0 or 1

If 1, clears all the previously specified control files. This allows a subsequent arg-file to ignore what has preceded it and specify new control files from scratch.

--clear-case 0 or 1

If 1, clears all the previously specified case files. This allows a subsequent arg-file to ignore what has preceded it and specify new case files from scratch.

--exclusion-list filename or --

Reads exclusions a line at a time and passes them unchanged to scoreassoc. If instead of  a filename -- is specified then the exclusions will be read from the current file until the end of file is reached.

Options to be used by other programs

Related programs use some of the same options as geneVarAssoc but also some specific to themselves. These programs will be documented subsequently.

--test-name name

Specify identifier for analysis to be run by intVarAssoc.

--interval-list-file file

Specifies file containing list of intervals from which intVarAssoc will extract variables and perform a scoreassoc analysis. Intervals are in format chr:start-end, e.g. 5:140000-240000.

--position chr:pos

A coordinate is given consisting of a chromosome followed by a colon and then base position, to be used by related programs, not geneVarAssoc itself.
